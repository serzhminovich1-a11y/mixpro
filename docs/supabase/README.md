# MIXPRO LMS — миграции БД (этап 1: архитектура)

Это расширение текущего Supabase-проекта MIXPRO под систему курсов,
достижений, XP/уровней, портфолио, заданий и сертификации звукорежиссёров.
Существующие таблицы `profiles` и `scores` не удаляются — `profiles` только
дополняется новыми колонками.

## Как применить

**Самый простой способ:** Supabase Dashboard → SQL Editor → New query →
открой `migrations/000_ALL_IN_ONE.sql`, скопируй весь файл целиком, вставь
и нажми Run. Это все три шага ниже, объединённые в один — порядок внутри
файла уже правильный.

Либо, если хочешь применять по частям (проще найти ошибку, если что-то
пойдёт не так) — выполни отдельно, **строго по порядку**:

1. `migrations/001_lms_schema.sql` — типы, новые таблицы, колонки в `profiles`,
   функция `get_level_from_xp()`.
2. `migrations/002_lms_rls.sql` — политики безопасности (Row Level Security)
   на все новые таблицы + защита `role`/`xp`/`verification_status` от прямого
   изменения пользователем.
3. `migrations/003_lms_seed.sql` — 4 примера достижений для проверки.

Если запрос упадёт с ошибкой — разберись сначала с ней, прежде чем
продолжать (частая причина: файл/блок 001 не выполнился полностью).

## Как проверить, что всё встало

В SQL Editor выполни по очереди:

```sql
-- 1. Таблицы появились
select table_name from information_schema.tables
where table_schema = 'public' and table_name in
  ('courses','lessons','lesson_progress','achievements','user_achievements',
   'projects','assignments','assignment_submissions','reviews',
   'certificates','verification_requests');
-- должно вернуть все 11 строк

-- 2. Функция уровня работает
select get_level_from_xp(0), get_level_from_xp(600), get_level_from_xp(6000);
-- Beginner | Intermediate | Professional

-- 3. Сид-достижения на месте
select title, condition_type, condition_value, xp_reward from achievements;
-- 4 строки

-- 4. У profiles появились новые колонки
select role, xp, bio, avatar_url, verification_status from profiles limit 1;
```

Проверка RLS (защита от самостоятельной накрутки XP/роли) — выполни как
залогиненный обычный пользователь (не через service_role):

```sql
update profiles set xp = 99999 where id = auth.uid();
```

Должна вернуться ошибка `role/xp/verification_status можно менять только
через ADMIN или service_role` — это подтверждает, что триггер защиты работает.

## Этап 2 — автовыдача достижений (сделано)

Файл `migrations/004_achievement_engine.sql` — та же процедура: открыть
Supabase → SQL Editor → New query → вставить весь файл → Run.

Что он делает: раньше достижения просто лежали в базе, никто их не выдавал.
Теперь, когда пользователь проходит урок / сдаёт задание / загружает проект,
база сама проверяет — не заслужил ли он награду — и если да, выдаёт её и
начисляет очки опыта. Это происходит автоматически внутри Supabase, без
участия сайта.

Отдельно проверять не обязательно — это станет видно само, когда появится
экран профиля с достижениями (этап 3).

## Этап 3 — портфолио (загрузка аудио)

Файл `migrations/005_storage_portfolio.sql` — открыть Supabase → SQL Editor →
New query → вставить весь файл → Run.

Что он делает: создаёт в Supabase хранилище файлов с названием "portfolio"
и правила доступа — каждый может загружать и слушать свои файлы, но удалять
может только тот, кто загрузил.

На сайте появилась страница "Портфолио" (ссылка есть в меню и в профиле) —
там можно загрузить свой микс (название + файл), он появится в списке
и станет виден на странице профиля через достижение "Первый микс".

Проверить: зайди на страницу профиля → "Портфолио" → загрузи любой mp3/wav
файл → должен появиться в списке ниже с плеером.

## Этап 3 — курсы и уроки (защищённое видео)

Файл `migrations/006_storage_lessons.sql` — открыть Supabase → SQL Editor →
New query → вставить весь файл → Run.

Что он делает: создаёт отдельное хранилище для видео уроков. В отличие
от портфолио, оно **закрытое** — видео нельзя открыть по прямой ссылке,
только через временную ссылку (час), которую генерирует сайт для
залогиненного пользователя. Плюс на видео нет кнопки "скачать", а поверх
идёт полупрозрачная подпись с именем смотрящего — если видео всё же
запишут с экрана и выложат, будет видно, чей это аккаунт. Это не
"невозможно скачать" (так не умеет никто, даже Netflix), а разумный
барьер от обычного распространения.

На сайте появилась страница **"Курсы"** (ссылка в меню) — там видно все
курсы и уроки внутри них, с отметкой "пройдено/не пройдено".

**Чтобы самому загружать курсы и уроки**, нужно один раз сделать свой
аккаунт преподавателем (роль ADMIN). Это делается вручную в SQL Editor —
специально так, чтобы никто не мог выдать себе права сам через сайт.

1. Найди свой `id`: `select id, username from profiles where username = 'ТВОЙ_НИК';`
   (скопируй значение id из результата).
2. Выполни (подставь свой id вместо `ВСТАВЬ_ID`):
   ```sql
   select set_config('app.bypass_privileged_guard', 'true', true);
   update profiles set role = 'ADMIN' where id = 'ВСТАВЬ_ID';
   ```
3. Обнови страницу сайта — в меню появится "Панель управления", там можно
   создавать курсы и добавлять к ним видео-уроки. Там же теперь есть кнопки
   редактировать (✏️) и удалить (🗑) у каждого курса и урока.

## Этап 3 — подтверждение опыта (кто ещё может создавать курсы)

Файл `migrations/007_verification_workflow.sql` — открыть Supabase → SQL
Editor → New query → вставить весь файл → Run.

Раньше загружать курсы мог только тот, кому ты вручную дал роль ADMIN
через SQL. Теперь это происходит по-человечески: любой пользователь
может подать заявку "Стать автором курсов" (ссылка появилась в меню
рядом с "Курсы") — описывает свой опыт, ты как ADMIN/MENTOR видишь эту
заявку в панели управления (раздел **"Верификация"**) и одним нажатием
подтверждаешь или отклоняешь. При подтверждении человеку автоматически выдаётся статус
VERIFIED_PRO, и он получает те же права на курсы, что MENTOR/ADMIN —
руками в SQL уже ничего делать не надо.

Твой аккаунт (уже ADMIN) как и раньше видит и может редактировать/удалять
абсолютно все курсы, независимо от того, кто их создал.

## Этап 3 — панель владельца (все пользователи)

Файл `migrations/008_admin_full_access.sql` — открыть Supabase → SQL
Editor → New query → вставить весь файл → Run.

Раньше даже ADMIN не мог менять чужие профили — база не пускала (RLS).
Теперь в панели управления есть раздел **"Пользователи"** (виден только
твоему ADMIN-аккаунту) — список вообще всех, кто зарегистрирован, и у
каждого можно прямо в таблице поменять роль, XP и статус верификации.
Сохраняется сразу при изменении, без отдельной кнопки "Сохранить".

## Этап 3 — удаление аккаунтов целиком

Это отдельный шаг, потому что удалить чей-то логин может только код,
который выполняется на сервере Supabase, а не в браузере (у браузера
специально нет для этого прав — иначе кто угодно мог бы удалить кого
угодно). Такой серверный код у Supabase называется Edge Function.

Поставить её можно прямо на сайте Supabase, ничего скачивать не нужно:

1. Зайди в Supabase Dashboard → в левом меню **Edge Functions**.
2. **Deploy a new function** → **Via Editor**.
3. В названии функции впиши ровно: `delete-user`
4. Стереть весь пример-заготовку и вставить вместо него содержимое файла
   `docs/supabase/functions/delete-user/index.ts` целиком.
5. Нажать **Deploy function**. Подожди секунд 20–30.

После этого в разделе "Пользователи" у каждого (кроме тебя самого)
появится кнопка 🗑 — удаляет аккаунт полностью: человек больше не сможет
зайти, а его профиль/очки/проекты удаляются из базы. Это нельзя отменить,
сайт переспросит перед удалением.

## Этап 3 — сдача и проверка заданий

Файл `migrations/009_assignments_workflow.sql` — открыть Supabase → SQL
Editor → New query → вставить весь файл → Run.

Что появилось:
- В разделе "Курсы и уроки" у каждого курса теперь можно добавлять задания
  (название, описание, требования, макс. баллы), не только уроки.
- На странице курса ученик видит задания и может "сдать" — выбирает один
  из своих загруженных проектов (из портфолио) как ответ.
- Раздел **"Проверка заданий"** в панели (у VERIFIED_PRO/MENTOR/ADMIN)
  — список сданных работ на проверке, можно прослушать, поставить оценку
  и написать отзыв, принять или отклонить.
- Ученик видит результат прямо на странице курса: статус, оценку и отзыв.

## Этап 3 — единая панель управления

Раньше "Управление курсами", "Все пользователи", "Проверка заявок" и
"Проверка заданий" были четырьмя отдельными страницами — переключаться
между ними и держать в голове, что где, было неудобно. Теперь это одна
страница **`admin.html`** ("Панель управления" в меню) с боковым меню:
Обзор (счётчики: сколько курсов, заявок, заданий на проверке — сразу
видно, где что-то ждёт внимания), Курсы и уроки, Проверка заданий,
Верификация, Пользователи. Разделы показываются по правам: у VERIFIED_PRO
— только курсы и задания, у MENTOR — плюс верификация, у ADMIN — все
пять, включая пользователей. Никакой отдельной миграции для этого не
нужно — только сами файлы сайта.

## Этап 3 — блог/соцсеть (текст и вложения)

Файл `migrations/010_blog_schema.sql` — открыть Supabase → SQL Editor →
New query → вставить весь файл → Run.

Новая страница **"Лента"** (в меню). Что там есть:
- Публикация постов — текст + вложение (видео или любой файл, картинка
  тоже подойдёт как "файл").
- Реакции эмодзи (🔥 👏 ❤️ 😂 💡 👎) — нажал ещё раз ту же — снимается.
- Комментарии текстом.
- Подписка на автора поста (кнопка "Подписаться" у чужих постов).
- Удалить свой пост может автор; ты как ADMIN можешь удалить любой
  (модерация).

Отдельной миграции не требуется — только та же `010_blog_schema.sql`.

## Этап 3 — голосовые сообщения

Кнопка 🎙️ появилась и в поле создания поста, и у каждого комментария —
запись идёт прямо в браузере (просит доступ к микрофону при первом
использовании), после остановки показывает проигрыватель для проверки
перед отправкой. Голосовое сообщение к посту работает как обычное
вложение (просто ещё один способ его создать), голосовой комментарий —
хранится в той же колонке `audio_url`, что была заранее заложена в
таблице комментариев.

Отдельной миграции тоже не требуется — таблицы и хранилище те же, что
для остального блога.

## Этап 3 — редактирование и удаление в блоге

Файл `migrations/011_blog_edit.sql` — открыть Supabase → SQL Editor →
New query → вставить весь файл → Run.

Что починил/добавил:
- У комментариев раньше вообще не было кнопки удаления в интерфейсе
  (хотя база это уже разрешала) — теперь есть.
- Свой пост и свой текстовый комментарий теперь можно редактировать
  (✏️ / "Изменить") — рядом появляется пометка "· изменено".
- Голосовые комментарии не редактируются (можно только удалить и
  записать заново) — редактировать звук "на месте" не имеет смысла.
- Ты как ADMIN можешь удалить любой пост и любой комментарий
  (модерация), но не можешь редактировать чужие — переписывать чужие
  слова от чужого имени не должно быть можно даже администратору.
